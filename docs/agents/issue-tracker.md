# Issue tracker: GitHub

Issues and PRDs for this repo live as GitHub issues. Use the `gh` CLI for all operations.

## Conventions

- Create, read, update, label, comment on, and close issues using `gh issue`.
- Infer the repository from the current Git remote.
- Pull requests are not used as a triage request surface.
- When a skill says "publish to the issue tracker," create a GitHub issue.
- When a skill says "fetch the relevant ticket," read the corresponding GitHub issue and its comments.
- Wayfinder maps and child tickets use GitHub issues, sub-issues, and native dependencies when available.
