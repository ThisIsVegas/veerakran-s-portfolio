# Domain Docs

Before exploring the codebase, engineering skills should read:

- `CONTEXT.md` at the repository root when present
- `docs/adr/` entries relevant to the work

This is a single-context repository. Missing domain files should not block work; domain-modeling workflows create them when needed.

Use terminology defined in `CONTEXT.md` consistently. Surface any conflict with an existing ADR instead of silently overriding it.
