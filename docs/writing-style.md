# Writing Style

The portfolio should sound like an experienced software engineer.

Avoid:

- buzzwords
- marketing language
- exaggerated claims
- "passionate"
- "innovative"
- "thrive"

Prefer:

- direct language
- factual descriptions
- concise paragraphs

Do not invent metrics.

Do not use em dashes.

Use British/International spelling where appropriate.

Sound like the author is explaining his work, not selling himself.
